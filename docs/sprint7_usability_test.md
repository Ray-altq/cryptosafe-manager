## Сценарии

1. Запустить приложение.
2. Выбрать или создать vault.
3. Ввести master password и открыть vault.
4. Добавить новую запись с логином, паролем, категорией и тегами.
5. Найти запись через поиск.
6. Скопировать пароль и дождаться очистки clipboard.
7. Заблокировать vault кнопкой Lock.
8. Разблокировать vault кнопкой Unlock.
9. Выполнить безопасный encrypted export.
10. Открыть журнал аудита и найти последние события.
11. Вызвать Panic mode через меню или hotkey.
12. Восстановиться после Panic mode через unlock.

## Найденные UX-проблемы

- Пользователям не всегда очевидно, что Lock скрывает данные, а Unlock требует повторный master password.
- Tray может быть спрятан Windows в скрытых значках, поэтому пользователь не сразу понимает, что приложение работает в фоне.
- Export воспринимается как рискованная операция, поэтому нужны безопасные значения по умолчанию и понятные пояснения.
- Panic mode понятен после объяснения, но без подсказки пользователь может не знать про hotkey.
- Журнал аудита полезен, но обычному пользователю сложно понять технические названия событий.

## Улучшения

- Добавлена кнопка Unlock рядом с Lock.
- Добавлены scroll/touchpad-friendly диалоги для длинных форм.
- Экспорт по умолчанию ориентирован на защищенные форматы.
- Добавлен tray quick search и расширенное tray menu.
- Добавлена panic activation через hotkey, tray и mouse shake gesture.
- Добавлены понятные security status/notification тексты.
- Добавлена accessibility metadata для диалогов и keyboard shortcuts summary.
