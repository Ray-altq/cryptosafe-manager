# Руководство пользователя CryptoSafe Manager

Документ описывает основные действия в приложении с точки зрения обычного пользователя: как открыть vault, работать с записями, копировать данные, пользоваться аудитом, импортом, экспортом и безопасным обменом.

## 1. Запуск и выбор vault

1. Запустите приложение через `python run.py` или через собранный `CryptoSafe Manager.exe`.
2. В стартовом окне выберите существующий vault-файл или создайте новый.
3. После выбора vault введите мастер-пароль.

Если vault новый, приложение предложит первичную настройку мастер-пароля. Если vault уже существует, данные не перезаписываются: сначала выбирается файл, затем выполняется вход.

## 2. Мастер-пароль и блокировка

Мастер-пароль нужен для доступа к зашифрованному vault. Без него приложение не расшифровывает записи.

Основные действия:

- `Lock` блокирует vault и очищает активные секреты из состояния приложения.
- `Unlock` повторно запрашивает мастер-пароль и возвращает доступ к записям.
- Auto-lock срабатывает при неактивности, сворачивании или потере фокуса, если это включено в настройках.

## 3. Работа с записями

В главном окне доступны действия:

- `Добавить`: создать новую запись.
- `Изменить`: изменить выбранную запись.
- `Удалить`: удалить выбранную запись.
- `Показать пароль`: временно показать пароль выбранной записи.
- `Скопировать пароль`: скопировать пароль через защищенный clipboard-сервис.
- `Скопировать логин`: скопировать имя пользователя.
- `Скопировать запись`: скопировать данные записи в безопасном представлении.

В форме записи можно заполнить:

- название;
- имя пользователя;
- пароль;
- URL;
- заметки;
- категорию;
- теги;
- дополнительные поля.

Если окно формы не помещается на экран, его можно прокручивать колесом мыши или тачпадом.

## 4. Генератор паролей

При создании или изменении записи можно сгенерировать пароль. По умолчанию приложение предлагает защищенные параметры, чтобы пользователь не подбирал криптографические настройки вручную.

Рекомендуется использовать:

- длину не меньше 16 символов;
- буквы разных регистров;
- цифры;
- специальные символы;
- исключение неоднозначных символов, если пароль нужно вручную перепечатывать.

## 5. Поиск и фильтры

Поиск работает по записям vault и поддерживает фильтры:

- текстовый поиск;
- категория;
- тег;
- дата обновления;
- сила пароля.

Кнопка `Сбросить` очищает фильтры и возвращает полный список записей.

## 6. Clipboard и временное копирование

Clipboard-сервис нужен для безопасного копирования паролей и логинов.

Что делает приложение:

- ограничивает время хранения данных в буфере;
- очищает буфер после таймера;
- отслеживает внешние изменения clipboard;
- показывает предупреждения при проблемах очистки;
- поддерживает memory-only режим, где секрет не попадает в системный clipboard.

Если приложение сообщает, что clipboard очищен не полностью, лучше вручную очистить буфер или заблокировать vault.

## 7. Журнал аудита

Журнал аудита показывает важные события:

- запуск приложения;
- вход и выход;
- блокировка и разблокировка vault;
- создание, изменение и удаление записей;
- операции clipboard;
- импорт и экспорт;
- создание share-пакетов;
- проверки целостности журнала.

В окне аудита можно:

- фильтровать события;
- обновлять список;
- проверять целостность цепочки;
- экспортировать журнал в JSON, CSV, PDF или CEF.

Если свежие события не видны, нажмите `Обновить`. Новые события пишутся в активный vault-файл.

## 8. Импорт и экспорт vault

Экспорт vault нужен для резервного копирования или переноса данных.

Основные варианты:

- `Encrypted JSON`: защищенный формат CryptoSafe, подходит для резервной копии и обратного импорта.
- `Bitwarden encrypted JSON`: защищенный формат для переноса в Bitwarden без plaintext-файла на диске.
- `LastPass/CSV`: plaintext-форматы считаются небезопасными и требуют явного подтверждения, если включены для совместимости.

Рекомендуемый вариант по умолчанию: зашифрованный JSON.

При импорте приложение:

- проверяет формат;
- валидирует структуру;
- очищает потенциально опасный HTML/скрипты;
- показывает preview;
- импортирует записи только после подтверждения.

## 9. Безопасный обмен записью

Функция `Share` позволяет передать одну выбранную запись без раскрытия всего vault.

Поддерживаемые сценарии:

- зашифрованный файл с паролем;
- public-key encryption через RSA/ECC;
- QR/key exchange для обмена публичным ключом;
- ограничения по времени и правам.

Перед созданием share-пакета нужно выбрать запись. Если запись не выбрана, приложение должно предупредить сразу, до выбора параметров.

## 10. QR и ключи

Раздел `QR / Ключи` используется для обмена публичными ключами.

Типовой сценарий:

1. Один пользователь генерирует key exchange payload.
2. Приложение показывает QR прямо в окне и сохраняет файлы при необходимости.
3. Второй пользователь импортирует публичный ключ.
4. После этого можно отправлять зашифрованные share-пакеты.

Private key нельзя отправлять другим людям. Его нужно хранить отдельно.

## 11. Panic mode и tray

Panic mode нужен для быстрой защиты в стрессовой ситуации:

- блокирует vault;
- очищает clipboard;
- скрывает окно;
- пишет событие в аудит.

Tray-иконка позволяет быстро вернуть окно, заблокировать vault, выполнить quick search или выйти из приложения.

## 12. Резервное копирование

Для резервной копии используйте encrypted export. Не храните plaintext CSV/JSON с паролями на рабочем столе или в папке загрузок.

Рекомендуемый порядок:

1. Сделать encrypted JSON export.
2. Проверить импорт этого файла в тестовый vault.
3. Сохранить файл в защищенное место.
4. Не передавать мастер-пароль вместе с файлом.

## 13. Частые проблемы

Если приложение не запускается:

- проверьте, что зависимости установлены через `pip install -r requirements.txt`;
- запустите `python run.py` из корня проекта;
- для QR-функций проверьте наличие `segno`.

Если не видны кнопки в окне:

- увеличьте окно;
- используйте прокрутку колесом мыши или тачпадом;
- проверьте масштабирование Windows.

Если не работает tray:

- убедитесь, что установлен `pystray`;
- проверьте системную область уведомлений Windows;
- перезапустите приложение после установки зависимостей.
